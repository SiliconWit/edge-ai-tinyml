The IoT Systems course taught you to send sensor data to the cloud for processing, visualization, and alerting. But not every application can tolerate the latency, bandwidth cost, or privacy exposure of cloud round-trips. Edge AI puts the intelligence directly on the microcontroller. A sensor node that classifies vibration patterns locally, detects a wake word without an internet connection, or spots an anomaly in real time before uploading anything is a fundamentally different kind of embedded device. This course teaches you to build those devices. #TinyML #EdgeAI #EmbeddedML

## Why Edge AI on Microcontrollers?

Cloud ML is powerful, but it requires connectivity, introduces latency, consumes bandwidth, and raises privacy concerns. TinyML solves these problems by running inference directly on the MCU.

<CardGrid>
  <Card title="Real-Time Response" icon="star">
    Inference on-device takes milliseconds, not the hundreds of milliseconds (or seconds) required for a cloud round-trip. Wake word detection, gesture recognition, and anomaly alerts happen instantly. Lesson 5 builds a keyword spotter that responds in under 200 ms on an ESP32.
  </Card>

  <Card title="No Connectivity Required" icon="setting">
    An MCU running a TinyML model works in basements, remote fields, aircraft, and anywhere else without reliable internet. The model is baked into the firmware. No Wi-Fi, no cellular, no dependency on external servers.
  </Card>

  <Card title="Privacy by Design" icon="approve-check">
    Audio, images, and motion data never leave the device. The raw signal is processed on-chip and only the classification result (a label, a score, a flag) is transmitted. This is a hard requirement in medical, industrial, and consumer applications.
  </Card>

  <Card title="Microwatt Power Budgets" icon="warning">
    A Cortex-M4 running inference at 1 Hz draws single-digit milliamps. Compare that to a Wi-Fi radio transmitting raw sensor data continuously. TinyML enables battery-powered deployments lasting months or years.
  </Card>
</CardGrid>

## Course Structure

Each lesson follows a consistent cycle:

<Steps>
1. **The ML Problem**
   A concrete classification, detection, or anomaly task that benefits from on-device inference rather than cloud processing.

2. **Data Collection and Labeling**
   How to gather training data from real sensors, label it correctly, and split it into train/validation/test sets.

3. **Model Training**
   Train a neural network using TensorFlow, Edge Impulse, or scikit-learn. Understand the architecture choices that make a model small enough for an MCU.

4. **Optimization and Deployment**
   Quantize the model, convert it to TensorFlow Lite Micro format, embed it in firmware, and run inference on the target MCU.

5. **Evaluation and Integration**
   Measure accuracy, latency, and memory usage on real hardware. Integrate the inference result into application logic (alerts, actuators, display, MQTT publish).
</Steps>

## Getting Started

<Steps>
1. **Install Python 3.9+ and TensorFlow.** TensorFlow is used for model training on your PC. The TFLite Micro runtime is compiled into the MCU firmware separately.

2. **Create a free Edge Impulse account.** Go to [edgeimpulse.com](https://edgeimpulse.com) and register. Edge Impulse provides a web-based pipeline for data collection, training, and deployment. The free tier supports everything in this course.

3. **Ensure you have an ESP32 dev board.** Lesson 1 starts with the ESP32. If you completed the ESP32 course, you already have one.

4. **Get an MPU6050 accelerometer module.** Lessons 2, 6, and 7 use this sensor for motion and vibration data. Available for under 3 USD from most electronics suppliers.

5. **Start with Lesson 1.** The sine wave model walkthrough gives you the complete mental model of the TinyML deployment pipeline.
</Steps>

:::tip[You do not need a GPU]
All model training in this course runs on a CPU in minutes. The models are intentionally small (kilobytes, not gigabytes) because they must fit on microcontrollers. Edge Impulse handles training in the cloud if you prefer not to install TensorFlow locally.
:::
